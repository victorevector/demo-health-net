About

This section explains the mock scenario that informs the code base's assumptions.

Hospital X sends its lab orders to Lab X for processing. Hospital X's Electronic Health Record (EHR) system speaks in HL7 v.2.3, and Lab X's Lababoratory Information System (LIS) speaks in FHIR v.4.0.0. Both services require Mirth Connect to broker messages between the two.

Dr. Will Tickel works at Hospital X and orders an Urinalysis for his patient, Chris P. Bacon. The EHR system processes the order event and sends it to the appropriate Mirth channel. The Mirth channel transforms it and sends it to the LIS. The LIS requires that the message contain:

  1. Patient information to verify patient's identity during check in.
  2. The order.
  3. An identifier for the order.

Sometime later, the LIS processes an order completion event and sends the observations to the appropriate Mirth channel. The Mirth channel transforms it and sends it to the EHR. The EHR requires that the message contain:

  1. The original identifier to reconcile the observations with the original request.
  2. The observations.
